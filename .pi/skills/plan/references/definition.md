## Definition Lens

Use this reference when an epic needs executable tasks and acceptance criteria.

Focus on:

- decomposing work into agent-executable tasks that usually fit in one pass
- giving each task a concrete description with a single clear output
- writing 2-5 binary acceptance criteria per task
- rejecting vague wording such as "reasonable", "appropriate", or "as needed"
- splitting tasks that hide multiple independent decisions

If you cannot write concrete acceptance criteria, the task is still underspecified.
